unitED

$ a team-built unit conversion calculator

overview

A collaborative software engineering project: a fully functional unit conversion calculator built in Java using Scrum methodology. Features include unit conversions, custom unit creation, calculation history, and a help section.

My primary focus was the conversion engine and GUI components. The project emphasized collaborative development practices — version control, task assignment via Scrum board, and iterative delivery.

tech stack

  • Java — JavaFX & Swing, OOP
  • JUnit — unit testing
  • Git & GitHub — version control
  • Scrum — agile development methodology

challenges & solutions

Most of my personal challenges came from working in a team for the first time at this scale. Three friction areas stood out:

Version control: Several members were new to Git, which led to merge conflicts and lost work. We ran a short training session early on — it helped, but merge issues still surfaced throughout the project.

Task dependencies: When upstream tasks slipped, downstream work stalled. We adjusted by restructuring assignments to minimize hard dependencies where possible.

Design consistency: Decisions were distributed across the team, resulting in an inconsistent UI and code style. Looking back, the codebase design should have been a shared, ongoing conversation rather than left to whoever owned the task.

takeaways

Team projects have a different failure mode than solo work. Technical debt accumulates faster when design decisions are distributed without coordination. Some key takeaways:

  • Teamwork and communication are prerequisites, not extras
  • Consistent code style requires proactive agreement, not assumed
  • Git fluency across the whole team is critical before you start
  • JavaFX GUI development and event-driven architecture
