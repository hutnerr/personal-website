navsys

$ a real-time GPS navigation system

overview

NavSys is a Java-based personal navigation system that uses a USB satellite receiver for real-time GPS updates. It parses GPGGA sentences from the satellite feed and displays live position on a map rendered in JavaFX.

Pathfinding uses Dijkstra's and Bellman-Ford algorithms on a graph of road nodes. The system supports route re-calculation and live position tracking as you move.

tech stack

  • Java - JavaFX, multithreading, MVC pattern
  • GPS hardware - USB satellite receiver, GPGGA sentence parsing
  • Pathfinding - Dijkstra's and Bellman-Ford on graph structures

challenges & solutions

Going in I knew nothing about GPS technology - I learned about satellite receivers, NMEA sentences, and coordinate systems while simultaneously building the system.

The biggest technical challenge was real-time performance. GPS data comes in continuously and the map must update smoothly. Threading was critical: the GPS read loop runs on its own thread and publishes position updates that the UI thread consumes without blocking.

takeaways

This project spanned hardware integration, real-time data processing, and algorithm implementation - a much broader scope than typical academic projects. Some key takeaways:

  • GPS hardware integration and NMEA sentence parsing
  • Multithreaded GUI design - producer/consumer with JavaFX
  • Practical application of graph algorithms for pathfinding
  • MVC, Observer, and Strategy patterns in a real system
