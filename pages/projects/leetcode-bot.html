beast[code]

$ a LeetCode Discord bot

overview

beast[code] is a Discord bot that facilitates LeetCode practice within a server. It runs a configurable system of recurring problems, contest reminder alerts, a point leaderboard, and user submission tracking — all driven by per-server and per-problem settings.

As a live service product it is constantly evolving. Server admins configure it via slash commands; users link their LeetCode accounts and submit solutions for points.

tech stack

  • Python — Discord.py, Requests
  • LeetCode API — GraphQL queries for problems and user data
  • Observer pattern for notification delivery

challenges & solutions

This was the largest project I had designed on my own at the time, which meant novel challenges at every stage. I built it in two phases: backend core first, then Discord integration — but I still found gaps in the backend mid-integration that required back-tracking.

One major scalability challenge was storage access. Initially I read from and wrote to files on every operation. I switched to an in-memory cache backed by JSON files, but this introduced cache/disk divergence bugs. I solved this with a "Synchronizer" that ensures both layers update atomically.

Getting the notification and timed problem delivery right required significant design work. I implemented an Observer pattern with time-bucketed scheduling: settings populate buckets, and a background loop fires notifications when the current time matches a bucket's threshold.

takeaways

Building and maintaining a live service product sharpened my understanding of system design, planning, and the real cost of shortcuts. Some key takeaways:

  • Designing and shipping a complete live service product end to end
  • Value of upfront planning — saved significant rework mid-integration
  • Cache/storage consistency as a first-class design concern
  • Maintaining and iterating on a deployed product over time
