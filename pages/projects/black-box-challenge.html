black box challenge

$ a terminal-based function guessing game

overview

The Black Box Challenge is a terminal-based coding game where players reverse-engineer mystery functions. You test the function with different inputs, observe the outputs, then write code to replicate its behavior using a custom programming language with its own lexer, parser, and interpreter.

Key features include a terminal UI with live code editing, a parameter testing table showing expected vs. actual outputs, real-time syntax error reporting, and support for custom mystery functions loaded from files.

tech stack

  • Custom language — lexer, parser, interpreter from scratch
  • Terminal.Gui for the terminal UI
  • Abstract Syntax Trees + Visitor Pattern
  • Recursive descent parsing, tree traversal

takeaways

Building a complete interpreter from lexer to evaluator illuminated how programming languages work under the hood and the value of clean design patterns. Some key takeaways:

  • Implementing lexers and parsers using formal grammars and BNF notation
  • Applying the visitor pattern to separate operations from data structures
  • Building and traversing abstract syntax trees for code evaluation
  • Managing operator precedence and associativity in expression parsing
  • Creating interactive terminal applications with Terminal.Gui
