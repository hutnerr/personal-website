obsidian attachment plugin

$ rule-based attachment placement for Obsidian.md

overview

An officially published Obsidian community plugin that gives you per-folder control over where new attachments are saved. Instead of dumping everything into one global folder, you define placement rules per directory - when you paste or drop a file, the plugin walks up the folder tree from the current note and applies the first matching rule.

Built out of personal frustration with Obsidian's single-destination attachment behavior in larger, organized vaults.

tech stack

  • TypeScript - Obsidian plugin API

how it works

When a new attachment is created, the plugin starts at the folder containing the active note and searches up the directory tree for a matching placement rule. The first rule found determines the destination. If no rule exists in the entire hierarchy, it falls back to a configurable default.

This means different areas of your vault can silently route attachments to different locations - no manual sorting needed after the fact.
